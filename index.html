MAD-EZ: Semi-Automated API Documentation

August 2016 - Present | Software Evolution Lab

Mad-ez, or Macro-based API Documentation Editor Z, is an Eclipse plugin that aims to make writing API documentation faster for Java programmers. The basic concept behind the project is the creation of pre-defined macro's, which when given parameters can be expanded to form full descriptive sentences for documentation. These expanded write-ups are then inserted into javadoc comments within the relevant file. Version 1.0.0 enables users to:

  • Create new macro's using the key binding Command-1, which will activate a wizard with options of macros and initial parameters for the user to choose

  • Edit an existing macro by pressing any key whilst the cursor is within an expanded macro. This activates a wizard which allows for users to change current parameters.

The project is currently in a private repository. If you are interested in the code and building process, feel free to contact me.

Stop Harassment

December 2016 - Present | Uplift

Stop Harassment is a Chrome Extension that removes harassing words in tweets and swaps them out for funny or affirming phrases. I was a part of the team who created the backbone of the app during the Lesbians Who Tech Hackathon, you can read more about the initial project here. I am currently working on the extension as a Winter volunteer. My first responsibility was to design the settings page of the extension, where the user can add or delete trigger words to filter.I then worked on a stemming solution, wherein the extension can detect different lexical forms of the trigger words to further filter. You can find the source code for the extension here.

OPAL

September 2016 - December 2016 | Health Informatics Group

OPAL is a hybrid phone app, built on Cordova and Angular, that is designed to improve the experience of oncology patients. The app is currently in it's trial use at the Royal Victoria Hospital in Montreal. An important part of the app is the ability to collect information from the patients through Patient Reported Outcomes, or PRO's, that are pushed to patient's phones periodically. This information is used to guide the consultation process. Over the course of this project, I designed and implemented views for PRO historical trends. This enables physicians to visually track the patients progress. My implementaton also included highlighting for data trends that were deemed unsafe and required attention. The report for this project can be read here. The project is currently in a private repository in order to protect patient privacy. If you'd like to know more about my code feel free to contact me.
